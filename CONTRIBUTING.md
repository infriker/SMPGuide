# Как внести вклад

Спасибо за интерес к проекту СМПГайд! Ниже описано, как можно помочь.

## Что можно улучшить

- Исправить или дополнить медицинские шаблоны карт вызова
- Добавить новые шпаргалки или калькуляторы
- Исправить ошибки в приложении
- Улучшить интерфейс

## Подготовка окружения

1. Установите [Android Studio Narwhal](https://developer.android.com/studio) (2025.1+)
2. Форкните и клонируйте репозиторий:
   ```bash
   git clone https://github.com/<ваш-username>/SMPGuide.git
   ```
3. Откройте проект в Android Studio и дождитесь Gradle Sync
4. Запустите приложение на устройстве или эмуляторе (минимум Android 7.0 / API 24)

## Как добавить контент

### Новый шаблон карты вызова

1. Создайте HTML-файл в `app/src/main/assets/templates/<категория>/`
2. Добавьте `MenuItem` в соответствующую категорию в `DataSource.kt`:
   ```kotlin
   MenuItem("Название шаблона", "templates/категория/файл.html")
   ```

### Новая шпаргалка

1. Создайте HTML-файл в `app/src/main/assets/cheatsheets/`
2. Добавьте `MenuItem` в список `cheatSheets` в `DataSource.kt`

### Новый калькулятор

1. Создайте HTML-файл в `app/src/main/assets/calculators/`
2. Добавьте `MenuItem` в список `calculators` в `DataSource.kt`

### Формат HTML-файлов

Файлы — обычный HTML. Можно использовать таблицы, списки, выделение текста. Файл будет отображён через WebView. Внешние зависимости не нужны — всё должно работать офлайн.

## Отправка изменений

1. Создайте ветку от `main`:
   ```bash
   git checkout -b feature/краткое-описание
   ```
2. Внесите изменения и сделайте коммит:
   ```bash
   git add .
   git commit -m "Добавлен шаблон: описание"
   ```
3. Отправьте ветку и откройте Pull Request:
   ```bash
   git push origin feature/краткое-описание
   ```

## Правила оформления коммитов

Используйте понятные сообщения на русском или английском:

- `Добавлен шаблон: Острый аппендицит`
- `Исправлена шпаргалка: Детские дозы`
- `Добавлен калькулятор: Шкала SOFA`
- `Исправлена навигация в боковом меню`

## Правила для Pull Request

- Один PR — одно логическое изменение
- В описании PR укажите, что именно изменено и почему
- Убедитесь, что проект собирается без ошибок

## Медицинский контент

При добавлении или исправлении медицинского контента:

- Указывайте источник информации (клинические рекомендации, приказы и т.д.)
- Убедитесь в актуальности данных
- Проверяйте дозировки и алгоритмы перед отправкой

## Нашли ошибку?

Откройте Issue с описанием проблемы:

- Что произошло
- Что ожидалось
- Версия Android и модель устройства (если баг в приложении)
- Скриншот (если применимо)
